超参数：

  • periods：控制报告的粒度

降低损失 (Reducing Loss)

发表于 2018-03-31

如何降低损失

迭代方法

不断迭代，直到总体损失不再变化或至少变化极其缓慢为止。这时候，我们可以说该模型已收敛。

梯度下降

凸形问题只有一个最低点；即只存在一个斜率正好为 0的位置。这个最小值就是损失函数收敛之处。

  • $(y-y’)^2$相对权重$w$和偏差$b$的导数表明样本的损失变化
  • 降低损失的方向上反复取小步（负梯度步长）
  • 梯度是偏导数的矢量；它可以让您了解哪个方向距离目标“更近”或“更远”。

学习速率

学习速率：步长。
超参数是编程人员在机器学习算法中用于调整的旋钮。

  1. 学习速率过低：费时。
  2. 学习速率过高：永远在U形曲线的底部随意弹跳。
  3. 金发姑娘学习速率：损失函数的梯度较小，则可以放心地试着采用更大的学习速率。

权重初始化

  • 凸形：权重从任意位置开始，只有一个最低点。
  • 非凸形：有多个最低点，很大程度上取决于初始值。

SGD 和小批量梯度下降法

批量指的是用于在单次迭代中计算梯度的样本总数。

  1. 超大批量，则单次迭代就可能要花费很长时间进行计算。
  2. 批量大小越大，出现冗余的可能性就越高。
  3. 冗余可能有助于消除杂乱的梯度，但超大批量所具备的预测价值往往并不比大型批量高。
  • 随机梯度下降法：一次抽取一个样本。
  • 小批量梯度下降法：每批包含 10-1000 个样本。

机器学习主要术语

发表于 2018-03-31

机器学习主要术语

(监督式）机器学习:通过学习如何组合输入信息来对从未见过的数据做出有用的预测.

标签:要预测的事物，即简单线性回归中的 y 变量。

特征:输入变量，即简单线性回归中的 x 变量。

样本：数据的特定实例：x。

  1. 有标签样本：同时包含特征和标签。用来训练模型。
    labeled examples: {features, label}: (x, y)

  2. 无标签样本：包含特征，但不包含标签。即：

    unlabeled examples: {features, ?}: (x, ?)

模型:定义特征与标签之间的关系。
生命周期两个阶段：

  • 训练：创建或学习模型。向模型展示有标签样本，让模型逐渐学习特征与标签之间的关系。
  • 推断：将训练后的模型应用于无标签样本。用训练后的模型来做出有用的预测 (y’)

回归与分类

  • 回归模型：预测连续值。
  • 分类模型：预测离散值。

标签：量化的指标。
特征：可量化的。

线性回归

发表于 2018-03-31

线性回归

标签（空格分隔）： 机器学习速成课程

线性回归

  1. 数据绘制成图表，了解下数据的分布
  2. 绘制一条直线来近似地表示这种关系。
    $y’$ 标签（理想输出），$w_1$特征1权重，$x_1$特征（已知输入），$b$偏差（截距）。
  3. 用新的$x_1$推断（预测）$y’$。
  4. 多个特征来表示更复杂的模型。

训练与损失

训练模型：通过有标签样本来学习（确定）所有权重和偏差的理想值
经验风险最小化：检查多个样本并尝试找出可最大限度地减少损失的模型。
损失：对于单个样本而言模型预测的准确程度的一个数值。
训练模型的目标：从所有样本中找到一组平均损失“较小”的权重和偏差。

平方损失（$L_2$ 损失）：一种常见的损失函数.

$= the square of the difference between the label and the prediction$
$= (observation - prediction(x))^2$
$= (y - y’)^2$

均方误差 (MSE):每个样本的平均平方损失.

  • $(x,y)$：样本.
    $x$：模型预测时的特征集。
    $y$：样本标签。
  • $prediction(x)$: 权重*特征集$x$+偏差的函数
  • $D$：多标签样本集$(x,y)$
  • $N$: $D$中样本个数。

‘Hexo配置'

发表于 2018-03-30 | 分类于 搭建博客

转载：https://www.cnblogs.com/fengxiongZz/p/7707219.html

1、安装Node.js+Git。
2、创建github.io
3、创建文件夹，安装Hexo
a、输入npm install hexo -g，开始安装Hexo
b、输入hexo -v，检查hexo是否安装成功
c、输入hexo init，初始化该文件夹
d、输入npm install，安装组件
f、输入hexo g，首次体验Hexo
g、输入hexo s，开启服务器
4、将Hexo与Github page绑定
a、文件夹里面鼠标右键，点击Git Base Here
5、配置Deployment，在其文件夹中，找到_config.yml文件，修改repo值（在末尾）
deploy:
type: git
repository: git@github.com:yourGit/yourGit.github.io.git
6、新建博客，hexo new post “博客名”
在文件夹_posts目录下将会看到已经创建的文件。
7、在生成以及部署文章之前，需要安装一个扩展：
npm install hexo-deployer-git —save
8、生成以及部署：hexo d -g
9、开启hexo：hexo s
链接 http://localhost:4000/.
10、查看博客：http://用户名.github.io
